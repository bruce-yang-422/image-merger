# 圖片合併工具開發文件

本文件用來記錄目前專案的實作狀態、架構分工與接下來的開發方向。以現在的進度來看，網頁版已經是主要版本，Python 版則維持在實用但精簡的 CLI 工具定位。

## 一、專案定位

本專案目前採雙版本架構：

- 網頁版：`HTML + CSS + JavaScript + Canvas`
- Python 版：`Python + Pillow`

實際產品策略如下：

- 網頁版負責互動操作、即時預覽、下載結果
- Python 版負責資料夾輸入輸出、批次處理與腳本整合

```text
https://image-merger.stack-base.com/
```

## 二、目前專案結構

```text
image-merger/
├─ index.html
├─ assets/
│  ├─ css/
│  │  ├─ style.css
│  │  ├─ foundation.css
│  │  ├─ components.css
│  │  ├─ responsive.css
│  │  └─ themes.css
│  └─ js/
│     ├─ app.js
│     └─ core.js
├─ cli.py
├─ requirements.txt
├─ doc/
│  └─ 開發文件.md
└─ src/
   └─ image_merger/
      ├─ __init__.py
      └─ merge.py
```

### 檔案用途

- `index.html`：網頁版主介面
- `assets/css/style.css`：CSS 匯入入口
- `assets/css/foundation.css`：基礎樣式與版面骨架
- `assets/css/components.css`：元件樣式
- `assets/css/responsive.css`：響應式規則
- `assets/css/themes.css`：淺色、深色與跟隨系統主題變數
- `assets/js/app.js`：前端模組入口
- `assets/js/core.js`：上傳、排序、預覽、合成、下載、主題切換等邏輯
- `cli.py`：Python 命令列入口
- `src/image_merger/merge.py`：Python 圖片合併核心

## 三、網頁版目前功能

### 已完成

- 圖片上傳
- 拖放上傳
- 最多 50 張圖片限制
- 支援 `JPG`、`PNG`、`WebP` 輸入
- 圖片列表顯示與縮圖
- 依檔名自然排序
- 拖曳排序
- 單張刪除與全部清除
- 排列模式：`vertical`、`horizontal`、`grid`
- 輸出尺寸預設按鈕與自訂數值
- 間距設定
- 背景色預設色票、自訂色票、文字輸入
- 輸出格式：`JPG`、`PNG`、`WebP`
- `JPG` 品質控制
- 圓角設定
- `DPI` 設定與實體尺寸換算
- 預覽區顯示
- 預估輸出大小資訊
- 分段輸出設定
- 高度警告與強制分段邏輯
- 多檔下載
- 主題切換：`System / Light / Dark`

### UI 現況

最近一輪前端調整已包含：

- 左右雙欄版面，右側固定預覽
- 主題切換按鈕改為英中雙語
- 背景色區塊改為常用色票與自訂色組合
- 設定區取消摺疊，改成固定展開
- 預覽區移除多餘佔位與過重外框
- 針對部分欄位寬度做細修，例如圓角自訂、DPI 自訂

### 仍可持續優化

- 更完整的空狀態與錯誤提示文案
- 行動版操作流暢度
- 更多輸出檔名策略
- 大尺寸圖片與大量圖片壓力測試
- 更精準的預估檔案大小算法

## 四、Python 版目前功能

### 已完成

- 使用 `--input-dir` 指定輸入資料夾
- 使用 `--output-dir` 指定輸出資料夾
- 使用 `--output-name` 指定輸出檔名主體
- 支援 `png`、`jpg`、`jpeg` 輸出
- 支援垂直與水平合併
- 支援間距、留白、背景色
- 支援遞迴搜尋圖片
- 會自動建立輸出資料夾
- 依檔名排序讀取圖片

### 目前限制

- 尚未支援 `Grid` 模式
- 尚未支援品質控制
- 尚未支援 `WebP` 輸出
- 尚未支援 `DPI` 設定
- 尚未支援分段輸出
- 沒有互動式預覽

### 目前定位

Python 版目前更適合作為：

- 批次合併工具
- 離線工具
- 自動化腳本的一部分
- 後續共用規格的參考版本
