# 圖片合併工具開發文件

本文件依照目前專案實際狀態整理，重點反映現有 HTML 網頁版的完成度，並補充 Python 版現況與後續待辦。  
截至目前，網頁版已接近可用版本；Python 版則仍維持基礎命令列功能。

---

# 一、目前專案定位

## 1. 產品方向

本專案採用雙版本架構：

- 網頁版：純前端 `HTML + CSS + JavaScript + Canvas`
- Python 版：本機命令列工具，使用 `Pillow`

## 2. 目前開發重點

現階段主力已轉向網頁版，目標是先完成一個可以直接部署到 GitHub Pages 的線上圖片合併工具。

目前判斷：

- HTML 版：接近完成
- Python 版：MVP 可用，但功能較精簡

# 四、HTML 版功能細節

## 1. 上傳機制

目前支援：

- 點擊選檔
- 拖放圖片到上傳區

規則：

- 僅接受 `jpg`、`jpeg`、`png`、`webp`
- 最多 50 張
- 載入失敗時顯示錯誤訊息

## 2. 圖片管理

目前圖片列表已支援：

- 顯示縮圖
- 顯示檔名
- 顯示原始尺寸
- 依檔名排序
- 拖曳改變順序
- 刪除單張
- 全部清空

這代表 HTML 版已超出最初 MVP，只差一些體驗細節可再收尾。

## 3. 拼接模式

目前實作中的合成模式：

- `vertical`
- `horizontal`
- `grid`

對應行為如下：

- 垂直模式：以輸出寬度為基準等比例縮放
- 水平模式：以輸出高度為基準等比例縮放
- Grid 模式：以固定 cell 尺寸置中排版

## 4. 輸出設定

目前支援：

- 格式：JPG / PNG / WebP
- JPG 品質控制
- 背景色設定
- 圓角設定
- DPI 設定

這裡和舊版規劃有一個明顯差異：

- 舊文件寫的是「不提供 WebP 輸出」
- 但目前 HTML 版實作中，已經支援 `WebP` 輸出

因此後續文件應以實作為準，不再沿用舊規格描述。

## 5. 預覽與尺寸資訊

目前預覽區已提供：

- Canvas 預覽
- 輸出寬高資訊
- 預估檔案大小
- 依 DPI 換算的實體尺寸

預覽策略為縮小渲染，不直接用 full size 畫面，可降低瀏覽器負擔。

## 6. 分段輸出

目前已實作分段輸出邏輯。

行為：

- 可手動啟用自動分段
- 可設定最大高度
- 當高度超過安全限制時會提示
- 當高度超過更高門檻時會強制啟用分段

這是目前網頁版最接近正式可用的重要功能之一。
